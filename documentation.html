1 Introduction

1.1 Purpose

Produsul "SQL Tutoring" este un site web destinat învățării utilizatorilor limbajului SQL. Acesta oferă o interfață prin care utilizatorii pot rezolva exerciții practice pentru a-și îmbunătăți cunoștințele și abilitățile în SQL.

Este important de menționat că "SQL Tutoring" se concentrează exclusiv pe furnizarea de exerciții și instrumente pentru învățarea limbajului SQL și nu include alte subiecte sau materiale de instruire. Acest SRS acoperă întregul sistem și toate subsistemele implicate în oferirea acestui serviciu de instruire online.

1.2 Document Conventions

Acest SRS respectă anumite convenții de redactare și formatare pentru a asigura o coerență și o claritate în întreaga sa prezentare. În acest sens, urmează un set de standarde și convenții generale, precum:

De exemplu, variabilele sunt folosite pentru a defini prioritățile și alte aspecte semnificative în cadrul documentului.

1.3 Intended Audience and Reading Suggestions

Acest document este destinat celor care doresc să învețe limbajul SQL, inclusiv elevilor, studenților la informatică și celor care doresc să-și îmbunătățească abilitățile în SQL. Pentru elevi și studenți, secțiunile relevante includ introducerea în SQL prin rezolvarea de exercitii și descrierea interfeței utilizatorului pentru rezolvarea exercițiilor.

2 Overall Description

2.1 Product Perspective

Aplicația web propusă este un produs nou și independent, creat pentru a facilita procesul de învățare a limbajului SQL pentru începători. Aceasta nu face parte dintr-o familie de produse existente și nu înlocuiește alte sisteme existente. În schimb, își propune să ofere o soluție completă și eficientă pentru elevi și alte persoane interesate să-și îmbunătățească cunoștințele în SQL. Aplicația web va opera ca o entitate autonomă, oferind funcționalități de administrare a întrebărilor SQL, de rezolvare a acestora și de gestionare a progresului utilizatorilor. Nu este un component al unui sistem mai mare și nu are interfețe directe cu alte sisteme externe.

2.2 Product Functions

  • Produsul trebuie să permită administrarea întrebărilor SQL, permițând administratorilor să adauge, editeze și șteargă întrebări din diverse capitole. De asemenea, administratorii ar trebui să poată atribui niveluri de dificultate pentru fiecare întrebare (ușoară, medie, grea).
  • Utilizatorii, după înregistrare, ar trebui să poată rezolva întrebările SQL disponibile pe platformă. După răspunsul la un număr de întrebări (20), utilizatorii trebuie să formuleze propria întrebare. Un serviciu web va fi responsabil pentru verificarea corectitudinii răspunsurilor utilizatorilor.
  • Exportul și importul de întrebări în formate deschise JSON trebuie să fie acceptate.
  • În plus, produsul ar trebui să genereze rapoarte privind clasamentele și statisticile legate de întrebările stocate în baza de date.

2.3 Design and Implementation Constraints

  • Nefolosirea Framework-urilor: Dezvoltatorii nu vor putea folosi framework-uri pentru dezvoltarea aplicației. Aceasta implică crearea de soluții personalizate pentru funcționalități precum gestionarea sesiunilor, rutarea URL-urilor și gestionarea bazelor de date.
  • Tehnologii Specifice: Dezvoltarea aplicației se va realiza folosind Node.js pentru partea de backend și MySQL pentru baza de date.

3 External Interface Requirements

3.1 User Interfaces

Interfața utilizatorului pentru aplicația noastră web de învățare a SQL-ului trebuie să fie intuitivă și ușor de utilizat, pentru a facilita procesul de învățare pentru utilizatori. Aceasta va consta din următoarele componente de software:

  • Interfața de Administrare: O interfață de administrare va permite administratorilor să adauge, să editeze și să șteargă întrebări SQL din diverse capitole. Această interfață va include formulare de introducere a întrebărilor și opțiuni de editare pentru a actualiza informațiile existente. Se va asigura că interfața de administrare respectă principiile de design UX/UI pentru a asigura o experiență de utilizare plăcută și eficientă.
  • Interfața Utilizatorului (UI): Interfața utilizatorului va permite utilizatorilor înregistrați să acceseze întrebările disponibile și să răspundă la acestea. Aceasta va include ecrane pentru vizualizarea întrebărilor disponibile, răspunsul la întrebări și formularea de întrebări proprii după cum este necesar. Se va asigura că interfața utilizatorului este simplă, curată și ușor de navigat, facilitând astfel procesul de învățare pentru utilizatori.

3.2 Software Interfaces

  • Bază de Date MySQL: Aplicația va comunica cu o bază de date MySQL pentru stocarea întrebărilor, a răspunsurilor utilizatorilor și a altor date relevante.
  • Node.js: Node.js va gestiona logica aplicației și va furniza servicii web pentru interacțiunea cu interfața utilizatorului și baza de date.
  • API (REST/GraphQL): Acest API va permite accesul la întrebările disponibile, rezolvările utilizatorilor și alte date relevante.
  • Tehnologii Frontend: HTML, CSS și JavaScript vor fi utilizate pentru a crea interfața utilizatorului pe partea de frontend.

4 System Features

Această secțiune prezintă cerințele funcționale ale produsului, organizate în funcție de caracteristicile majore ale sistemului. Aplicația noastră de învățare a limbajului SQL are ca scop facilitarea procesului de învățare pentru începători prin intermediul unei platforme online interactive. Astfel, funcționalitățile sistemului sunt structurate în jurul a două caracteristici principale:

4.1 Administrarea Întrebărilor SQL

Această caracteristică permite administratorilor să gestioneze întrebările disponibile pe platformă, inclusiv adăugarea, editarea și ștergerea acestora. Administratorii pot, de asemenea, atribui niveluri de dificultate pentru fiecare întrebare, contribuind astfel la adaptarea experienței de învățare la diferitele niveluri de cunoștințe ale utilizatorilor.

4.2 Rezolvarea și Formularea de Întrebări

Utilizatorii înregistrați au posibilitatea de a accesa întrebările disponibile și de a le rezolva. După ce au răspuns la un număr predefinit de întrebări, utilizatorii sunt încurajați să formuleze propriile întrebări, consolidând astfel cunoștințele dobândite și implicându-se activ în procesul de învățare.

5 Other Nonfunctional Requirements

5.1 Performance Requirements

  • Timp de Răspuns al Interfeței Utilizatorului: Interfața utilizatorului trebuie să ofere un timp de răspuns rapid, cu încărcarea paginilor sau a întrebărilor în mai puțin de ~2 secunde.
  • Gestionarea concurentei: Sistemul trebuie să fie capabil să gestioneze concurența a minimum ~100 de utilizatori simultan, fără a afecta performanța.

5.2 Safety Requirements

  • Protejarea datelor sensibile: oate datele sensibile ale utilizatorilor, cum ar fi informațiile de autentificare și răspunsurile la întrebări, trebuie să fie protejate prin criptare și să fie accesibile doar utilizatorilor autorizați.
  • Prevenirea atacurilor cibernetice: Sistemul trebuie să fie protejat împotriva atacurilor de tip Cross Site Scripting sau SQL injection.
